サーバのプロパティ

次の表はサーバがコンフィギュレーション設定で使うプロパティのリストです。

プロパティ 説明
java.rmi.server.codebase サーバの codebase URL を示します。クライアントはここからクラスをダウンロードできます。
java.rmi.server.disableHttp true にセットすると、RMI コールは HTTP を使用できなくなります。 この状態では、RMI はファイアウオール経由での呼び出しに HTTPを使いません。デフォルト設定は false です（HTTP を利用する）。
java.rmi.server.hostname 完全修飾したホスト名が InetAddress.getLocalHost() から得られないときに、ここに設定しておきます。デフォルトは非設定です。
java.rmi.dgc.leaseValue クライアントが VM のリモートオブジェクトを参照するためにクライアントに許される最大 lease 期間を設定します。デフォルト値は10分です。
java.rmi.server.logCalls true に設定すると、サーバコールのログが残され、標準エラー出力に表示されます。デフォルトでは false に設定されます。
java.rmi.server.useCodebaseOnly true に設定しておくと、RMI がクラスをローディングするときに（CLASSPATH が使えなければ）、プロパティ java.rmi.server.codebasede で指定される URL のみを参照してロードします。

その他のプロパティ

これらのプロパティは実装パッケージに含まれる特定の実装クラスが置かれている場所を指示するのに使われます。

プロパティ 説明
java.rmi.loader.packagePrefix インタフェース java.rmi.server.LoaderHandler を実装するクラスのためのパッケージプリフィックス。 デフォルト値は sun.rmi.server。
java.rmi.registry.packagePrefix インタフェース java.rmi.registry.RegistryHandler を実装するクラスのためのパッケージプリフィックス。 デフォルト値は sun.rmi.registry。
java.rmi.server.packagePrefix サーバパッケージのプリフィックス。前提として、サーバ参照クラスの実装 (UnicastRef や UnicastServerRef) はプリフィックスで指定されるパッケージに置かれる。デフォルト値は sum.rmi.server。
