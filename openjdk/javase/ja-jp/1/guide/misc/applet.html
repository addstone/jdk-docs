APPLET タグ

簡単な APPLET タグの例 : 


  <applet code="MyApplet.class" width=100 height=140></applet>
このタグは、ビューワまたはブラウザにアプレットをロードするように指示し、アプレットのコンパイル済みコードが MyApplet.class (現在の HTML ドキュメントと同じディレクトリ) 内にあることを示しています。また、アプレットの初期サイズを幅 100 ピクセル、高さ 140 ピクセルに設定しています。

少し複雑な APPLET タグの例 : 


  <applet codebase="http://java.sun.com/applets/applets/NervousText"
	code="NervousText.class" width=400 height=75>
  <param name="text" value="Welcome to HotJava!">
  <hr>
  Java 対応のブラウザ (HotJava) を使用すると、このパラグラフの代わりに踊るテキストが表示されます。
  <hr>
  </applet>
このタグは、ビューワまたはブラウザにアプレットをロードするよう指示し、アプレットのコンパイル済みコードが URL http://java.sun.com/applets/applets/NervousText/NervousText.class にあることを示しています。また、アプレットの初期サイズを 400x75 ピクセルに設定しています。 また、ビューワまたはブラウザは、アプレットの「text」属性 (このアプレットが表示するテキストをカスタマイズする属性) を「Welcome to HotJava!」に設定します。Java アプレットを表示することができないブラウザでこのページを表示した場合、ブラウザは APPLET タグと PARAM タグを無視し、<param> タグと </applet> タグの間にある HTML コード (アプレットの代わりに表示する HTML) だけを表示します。

Java 対応のブラウザ (HotJava) を使用すると、このパラグラフの代わりに踊るテキストが表示されます。

別の APPLET タグの例 : 


   <applet code=A21 width=256 height=256 archive="toir.jar">
   <param name=img value=test.gif>
   <hr>
   いくつかの標準アプレットをアーカイブを使用するように変換しなければなりません。
   誰かやりたい人はいますか ?
   <hr>
   </applet>
この例では、アプレットクラスは A21 です。このバイトコードは「toir.jar」というアーカイブ内に入っています。このアーカイブは、test.gif という名前のイメージリソースも含んでいる可能性があります (リソースについてのドキュメンテーションを参照)。

APPLET タグの完全構文を次に示します。必須エレメントはボールド体 で示します。任意指定のエレメントは通常の書体で示します。作成者が指定する必要のあるエレメントはイタリック体で示します。 

    <APPLET
        CODEBASE = codebaseURL
        ARCHIVE = archiveList
        CODE = appletFile ...or...  OBJECT = serializedApplet
        ALT = alternateText
        NAME = appletInstanceName
        WIDTH = pixels  HEIGHT = pixels
        ALIGN = alignment
        VSPACE = pixels  HSPACE = pixels
    >
    <PARAM NAME = appletAttribute1 VALUE = value>
    <PARAM NAME = appletAttribute2 VALUE = value>
    . . .
    alternateHTML
    </APPLET>

CODE、CODEBASE 等はアプレットタグの属性です。これらの属性は、アプレットについての情報をブラウザに伝えます。必須属性は、CODE、WIDTH、HEIGHT の 3 つだけです。以下では、各属性について詳しく説明します。

CODEBASE = codebaseURL
アプレットの基本 URL、またはアプレットのコードを含むディレクトリを表す任意指定の属性です。この属性を省略すると、ドキュメントの URL が使われます。
ARCHIVE = archiveList
事前にロードされるクラスおよびその他のリソースを含む、1 つまたは複数のアーカイブを表す任意指定の属性です。クラスは、指定の CODEBASE と AppletClassLoader のインスタンスを使用してロードされます。 archiveList 内には、アーカイブを「,」で区切って指定します。注意 : JDK1.1 では、複数の APPLET タグに同じ CODEBASE 属性を指定すると、ClassLoader の同じインスタンスを共有することになります。アプレット間通信を行うクライアントコードでは、この方法を使用します。将来の JDK では、別のアプレット間通信メカニズムが用意される * 可能性があります *。
CODE = appletFile
アプレットのコンパイル済み Applet サブクラスを含むファイルの名前を表す必須属性です。このファイルは、アプレットの基本 URL に対する相対名として解釈されます。絶対名は指定できません。CODE または OBJECT のどちらか一方を必ず指定します。
OBJECT = serializedApplet
Applet のシリアル化表現を含むファイル名を表す属性です。Applet は逆シリアル化されます。init() メソッドは * 呼び出されません * が、start() メソッドは呼び出されます。この属性は、元のオブジェクトがシリアル化されており、 * 復元されていない * 場合に有効となります。この APPLET インスタンスに渡されたすべての属性は、Applet で使用可能となります。この機能の使用方法には、とても厳しい制限があります。アプレットは、シリアル化する前に停止する必要があります。CODE または OBJECT のどちらか一方を必ず指定します。
ALT = alternateText
APPLET タグを認識するが、Java アプレットを実行できないブラウザでページを表示した場合に、表示するテキストを表す任意指定の属性です。
NAME = appletInstanceName
アプレットのインスタンス名を表す任意指定の属性です。同じページ上にあるアプレットは、この属性を使用してほかのアプレットをみつけ、互いに通信することができます。
WIDTH = pixels HEIGHT = pixels
アプレットの表示領域の初期サイズ (幅と高さ) を表す必須属性です。アプレットが表示するウィンドウやダイアログのサイズは含みません。
ALIGN = alignment
アプレットの配置を表す任意指定の属性です。この属性には、IMG タグと同じ値 (left、right、top、texttop、middle、absmiddle、baseline、bottom、absbottom) を指定することができます。
VSPACE = pixels HSPACE = pixels
アプレットの上下に空けるピクセル数 (VSPACE)、アプレットの左右に空けるピクセル数 (HSPACE) を表す任意指定の属性です。この属性は、IMG タグの VSPACE 属性および HSPACE と同じように扱われます。
<PARAM NAME = appletAttribute1 VALUE = value>
<PARAM NAME = appletAttribute2 VALUE = value> . . .
アプレット特有の属性を指定するためのタグです。アプレットは、getParameter() メソッドを使用して属性にアクセスします。
