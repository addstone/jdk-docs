セキュリティと署名付きアプレット

Java Security API は、低レベルおよび高レベルのセキュリティ機能を Java アプリケーションに組み込むことができるように設計されています。JDK1.1 の Java Security の最初のリリースでは、デジタル署名やメッセージダイジェストなど、この 機能のサブセットが含まれていましす。加えて、キー管理や証明管理、アクセスコントロー ルの abstract インタフェースがありました。X.509 v3 証明やその他の証明フォーマット、アク セスコントロール領域での豊富な機能をサポートする専用の API は、次の JDK リリースで提供します。

また、JDK 1.1 は、Java ARchive (JAR) ファイルに署名することのできるツールを提供 します。これにはクラスやその他のデータ（画像や音声など）を格納することができます。 アプレットビューワでは、信頼できるエントリによって（ツールを使用して）署名を付けら れたJAR ファイル内のダウンロードされたアプレットを、ローカルアプリケーションとまったく同じ 権限で実行することができます。つまり、この種のアプレットは、オリジナルの Java セキュリティモデルの「サンドボックス」制限の対象になりません。将来のリリー スで は、許容信頼レベルにおける細分性を強化するなど、より高度なセキュリティ原則を提供し ます。
